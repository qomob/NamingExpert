## 工作流程 / Workflow

### 1. 需求解析 / Requirement Analysis

从五个维度提取命名需求：命名类型、核心元素、风格偏好、语言/文化要求、约束条件。用户已提供的信息直接使用，仅对缺失的关键维度追问。

Extracts naming requirements across five dimensions: naming type, core elements, style preference, language/culture requirements, and constraints. Uses information already provided by the user, only asking follow-up questions for missing critical dimensions.

### 2. 创意生成 / Creative Generation

基于需求生成 **3–5 个**候选方案，每个方案包含：

- **字义解析 / Semantic Analysis**：每个字的含义与整体语义组合
- **文化象征 / Cultural Symbolism**：典故出处、文化内涵、象征意义
- **发音评估 / Phonetic Evaluation**：音韵节奏、谐音风险（含多语种）、朗朗上口程度

Generates **3–5 candidate proposals**, each analyzed from three dimensions: semantics, cultural symbolism, and phonetics (including cross-lingual homophone detection).

### 3. 智能优化 / Intelligent Optimization

- 风格匹配度评估 / Style match evaluation
- 商标/域名可用性提示 / Trademark & domain availability reminders
- 多语种歧义检测 / Cross-lingual ambiguity check
- 基于用户反馈的迭代优化 / Iterative refinement based on user feedback

## 命名方法论 / Naming Methodology

### 中文名策略 / Chinese Naming Strategies

- **诗词典故法**：从经典文献中提炼意象
- **声韵和谐法**：注重平仄搭配、韵母协调
- **会意组合法**：用字义叠加创造新意境
- **行业映射法**：将行业特征隐喻到名称中

### 英文名策略 / English Naming Strategies

- **词根组合法**：Latin/Greek 词根构建新词
- **缩写再造法**：取核心词首字母组合
- **隐喻联想法**：通过意象传达品牌精神
- **音韵模仿法**：创造发音悦耳的 neologism

### 多语言注意事项 / Multilingual Considerations

- 检查名称在主要语言中的歧义或负面含义
- 注意数字、颜色、动物等跨文化象征差异
- 日语命名需考虑汉字音读/训读差异
